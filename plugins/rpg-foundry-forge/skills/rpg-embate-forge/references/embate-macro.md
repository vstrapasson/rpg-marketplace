# Embate runner macro — the one-click live layer

The challenge journal is the spine; this macro is the **runnable** layer. It installs a single script macro on the GM hotbar that runs the whole `desafio`: it asks **who rolls** (GM rolls for the whole party, or the checks are sent to the players), rolls/collects each skill check, maps every **degree of success → Victory Points** using the `desafio`'s award table, and **advances the VP clock** journal page. It is the macro counterpart to the per-beat `request-player-rolls` calls — same rules, but self-contained and re-triggerable from one hotbar slot.

## How it ships

Generate the macro source by baking this challenge's data into the `CONFIG_EMBATE` block of the template below, then install it with the **`create-macro`** MCP tool (foundry-mcp ≥ 0.10.0):

```
create-macro
  name:       "Embate — «desafio name»"        // prefer-existing: re-running updates in place, no duplicate
  type:       "script"
  img:        "icons/svg/dice-target.svg"
  folderName: "Embates"                          // optional Macro-directory folder
  hotbarSlot: «1-50»                             // optional — global pin (every scene); ask the GM which slot
  sceneName:  "«local scene name»"               // optional — bind to the desafio's scene instead (≥ 0.11.0)
  flags:      { "foundry-mcp-bridge": { "challengeId": "«desafio name»" } }
  command:    "«the filled-in template below»"
```

Record the returned `macroId` in the manifest `desafio` row. A built challenge whose row already has a `macroId` is skipped on re-run (the tool updates by name regardless, so re-runs are safe).

**Scene-bound hotbar (≥ 0.11.0).** Pass `sceneName` (or `sceneId`) **instead of** a global `hotbarSlot` to pin the macro to the GM bar **only while that scene is viewed** — it clears when you leave. Use it when the `desafio` has a `local` whose scene the forge already ensured: bind the runner to that scene so it appears on the bar exactly when the GM opens the challenge's map, and never clutters other scenes. The slot is auto-picked from page 5 (slots 41+) unless you pass `hotbarSlot`. Mechanics: the tool writes the macro into the scene's `flags.foundry-mcp-bridge.hotbarMacros` (`{ "<slot>": "<macroId>" }`); a `canvasReady` hook in the module swaps those slots per viewed scene (the managed set is the union of all scenes' flags, GM-only). For a `desafio` with **no `local`** (most influence/research challenges), there is no scene to bind — use a global `hotbarSlot` or leave it in the `Embates` folder.

**Hard rules:**
- The macro runs **as the GM** (foundry-mcp installs it GM-owned). That is what lets it read any selected token, roll for any PC actor, and write the clock page — no per-actor ownership setup.
- **Bake real values only.** Pull `vpTarget`, `awards`, `beats` (skill + DC), `thresholds`, and the clock `journalId`/`pageId` from the resolved `desafio`. Never invent a DC or a skill the challenge didn't define — if a beat is missing, route back to `rpg-embate-builder` (same `missing[]` rule as the rest of the forge).
- For `scale: single-check`, set `clock: null` and a single entry in `beats` — the macro just rolls and reports, no VP track.
- Targets Foundry **v13–v14** (the bridge's compat range) and the **PF2e** system. `actor.skills[slug].roll(...)` returns the degree of success directly (`roll.degreeOfSuccess`, 0–3); `DialogV2` is `foundry.applications.api.DialogV2`.

## The award map

The default mirrors `rpg-embate-builder` (and PF2e's VP guidance): **crit success +2 / success +1 / failure 0 / crit failure −1**. Override per challenge only if the `desafio` body says so (e.g. Research caps at +2/+1, a Chase awards +1 on any success). The keys are the `DegreeOfSuccess` indices: `3`=crit success, `2`=success, `1`=failure, `0`=crit failure.

## Template

Fill every `«…»`. Everything else is literal.

```javascript
/* Embate runner — «desafio name» (generated by rpg-embate-forge) */
const MODULE_ID = "foundry-mcp-bridge";
const CONFIG_EMBATE = {
  name: "«desafio name»",
  subsystem: "«influence|chase|research|infiltration|generic»",
  trait: "«action:influence»",          // extraRollOption; "" if the scene has none
  vpTarget: «vp_target»,                 // e.g. 8 ; ignored when clock is null
  awards: { 3: 2, 2: 1, 1: 0, 0: -1 },   // DegreeOfSuccess → VP delta
  thresholds: [                          // crossing one fires its note (fail-forward / reward)
    { at: «n», note: "«what happens when VP reaches n»" },
  ],
  beats: [                               // the approaches the scene invites — NOT a closed list
    { label: "«Diplomacia — apelo à honra»", skill: "diplomacy",   dc: «25» },
    { label: "«Intimidação — ameaça velada»", skill: "intimidation", dc: «27» },
  ],
  clock: { journalId: "«journalId»", pageId: "«clockPageId»" }, // or null for single-check
};

// ---- helpers ---------------------------------------------------------------
const DialogV2 = foundry.applications.api.DialogV2;
const isGM = game.user.isGM;
const C = CONFIG_EMBATE;
const bar = (f, t) => "▰".repeat(Math.max(0, f)) + "▱".repeat(Math.max(0, t - f));
const clockPage = () => C.clock ? game.journal.get(C.clock.journalId)?.pages.get(C.clock.pageId) : null;

async function getVP() {
  const p = clockPage();
  if (!p) return 0;
  return Number(p.getFlag(MODULE_ID, "vp") ?? 0);
}
async function setVP(n) {
  const p = clockPage();
  if (!p) return n;
  const clamped = Math.max(0, Math.min(C.vpTarget, n));
  await p.setFlag(MODULE_ID, "vp", clamped);
  await p.update({ "text.content":
    `<p><strong>${C.name}</strong> — Victory Points</p><p style="font-size:1.4em">${bar(clamped, C.vpTarget)} ${clamped}/${C.vpTarget}</p>` });
  return clamped;
}
function gmWhisper(html) {
  return ChatMessage.create({ content: html, whisper: ChatMessage.getWhisperRecipients("GM").map(u => u.id) });
}
async function applyDelta(delta, lines) {
  if (!C.clock) { await gmWhisper(`<p><strong>${C.name}</strong></p>${lines.join("")}`); return; }
  const before = await getVP();
  const after = await setVP(before + delta);
  const crossed = C.thresholds.filter(t => before < t.at && after >= t.at)
    .map(t => `<p>⚑ <strong>Limiar ${t.at}:</strong> ${t.note}</p>`).join("");
  const done = after >= C.vpTarget ? `<p>🏁 <strong>Embate vencido</strong> (${after}/${C.vpTarget}).</p>` : "";
  await gmWhisper(`<p><strong>${C.name}</strong> ${bar(after, C.vpTarget)} ${after}/${C.vpTarget} (${delta >= 0 ? "+" : ""}${delta})</p>${lines.join("")}${crossed}${done}`);
}

// ---- pick mode + beat ------------------------------------------------------
const mode = await DialogV2.wait({
  window: { title: `Embate — ${C.name}` },
  content: `<p>Como esta rodada é resolvida?</p>`,
  rejectClose: false,
  buttons: [
    { action: "gm",      label: "🎲 GM rola por todos (tokens selecionados)" },
    { action: "players", label: "📨 Mandar pros jogadores (cada um rola)" },
    { action: "cancel",  label: "Cancelar" },
  ],
});
if (!mode || mode === "cancel") return;

const beatChoice = await DialogV2.wait({
  window: { title: `Embate — ${C.name}` },
  content: `<p>Qual abordagem nesta rodada?</p><select name="beat" style="width:100%">
    ${C.beats.map((b, i) => `<option value="${i}">${b.label} — CD ${b.dc}</option>`).join("")}</select>`,
  rejectClose: false,
  buttons: [{ action: "ok", label: "Confirmar", default: true,
    callback: (event, button, dialog) => { const f = button?.form ?? dialog?.element; return f?.querySelector?.("[name='beat']")?.value ?? "0"; } }],
});
if (beatChoice === null) return;
const beat = C.beats[Number(beatChoice)];
const extraRollOptions = C.trait ? [C.trait] : [];

// ---- branch A: GM rolls for every selected token ---------------------------
if (mode === "gm") {
  const tokens = canvas.tokens.controlled.filter(t => t.actor);
  if (!tokens.length) return ui.notifications.warn("Selecione os tokens dos PJs que vão rolar.");
  let total = 0; const lines = [];
  for (const tok of tokens) {
    const stat = tok.actor.skills?.[beat.skill] ?? tok.actor.getStatistic?.(beat.skill);
    if (!stat) { lines.push(`<p>⚠️ ${tok.name}: sem perícia “${beat.skill}”.</p>`); continue; }
    const roll = await stat.roll({ dc: { value: beat.dc }, extraRollOptions, skipDialog: true });
    if (!roll) continue;
    const dos = roll.degreeOfSuccess ?? 1;
    const delta = C.awards[dos] ?? 0;
    total += delta;
    const tier = ["falha crítica","falha","sucesso","sucesso crítico"][dos];
    lines.push(`<p>${tok.name}: <strong>${tier}</strong> (${roll.total} vs CD ${beat.dc}) → ${delta >= 0 ? "+" : ""}${delta} VP</p>`);
  }
  await applyDelta(total, lines);
  return;
}

// ---- branch B: send the checks to the players, then aggregate --------------
if (mode === "players") {
  const inline = `@Check[${beat.skill}|dc:${beat.dc}${C.trait ? "|traits:" + C.trait : ""}]`;
  await ChatMessage.create({ content:
    `<p><strong>${C.name}</strong> — ${beat.label}</p><p>Façam a rolagem: ${inline}</p>` });

  const tally = { vp: 0, lines: [] };
  const hookId = Hooks.on("createChatMessage", (msg) => {
    const ctx = msg.flags?.pf2e?.context;
    if (!ctx || ctx.outcome == null) return;
    const domains = ctx.domains ?? [];
    if (!domains.includes(beat.skill)) return;     // match this beat's skill
    const map = { criticalSuccess: C.awards[3], success: C.awards[2], failure: C.awards[1], criticalFailure: C.awards[0] };
    const delta = map[ctx.outcome] ?? 0;
    tally.vp += delta;
    tally.lines.push(`<p>${msg.alias ?? msg.speaker?.alias ?? "PJ"}: <strong>${ctx.outcome}</strong> → ${delta >= 0 ? "+" : ""}${delta} VP</p>`);
  });

  const finish = await DialogV2.wait({
    window: { title: `Embate — ${C.name}` },
    content: `<p>Aguardando as rolagens dos jogadores…</p><p>Clique <strong>Apurar</strong> quando todos rolarem.</p>`,
    rejectClose: false,
    buttons: [
      { action: "apply",  label: "Apurar e gravar VP", default: true },
      { action: "cancel", label: "Descartar rodada" },
    ],
  });
  Hooks.off("createChatMessage", hookId);
  if (finish === "apply") await applyDelta(tally.vp, tally.lines.length ? tally.lines : ["<p>Nenhuma rolagem capturada.</p>"]);
  return;
}
```

## Notes on the two branches

- **GM rolls for all** — fast montage pacing. Select the PJ tokens, run the macro, pick the approach; it rolls each token's skill, maps degrees to VP, sums, and ticks the clock. Run it again for a different approach/round. `skipDialog: true` keeps the batch snappy; drop it if the GM wants each modifiers dialog.
- **Send to players** — keeps each player's own dice and agency. The macro posts an `@Check` button to chat; players click and roll their own characters. A one-shot `createChatMessage` hook collects every matching check's `outcome` until the GM clicks **Apurar**, then writes the VP. The hook matches by `flags.pf2e.context.domains` including the beat's skill slug — **verified live on PF2e v8.1.2 / Foundry v14.363** (`context.domains` contains the slug, `context.outcome` is the degree string). If a future PF2e version reports the domain differently, widen the match (e.g. also test `ctx.options` for `action:<subsystem>`).
- **The clock is authoritative in a page flag** (`flags.foundry-mcp-bridge.vp`) and mirrored as a ▰▱ bar in the page text, so the GM sees it and the macro can advance it deterministically. This is why step 3 of the skill creates the VP clock as a *named page with an initial `vp` flag* — so the macro has a stable `pageId` to write. Threshold notes and the win line fire automatically as VP crosses them, whispered to the GM (fail-forward stays GM-side).

## Validation status

Exercised live on **PF2e v8.1.2 / Foundry v14.363** (via `create-macro` install + GM click), confirmed working: the `DialogV2` mode/approach prompts, **GM-rolls-all** (`actor.skills[slug].roll({dc})` → `degreeOfSuccess` → VP), **send-to-players** aggregation (the `createChatMessage` hook reading `flags.pf2e.context.{domains,outcome}`), threshold/win notes, and VP accumulation. **Not yet exercised:** the journal-**page** clock path (`page.setFlag` / `page.update`) — the live demo persisted VP in a GM-user flag to avoid creating a journal; the production page-clock variant above is unchanged and untested. Validate it the first time a real `desafio` is forged with a clock page.
