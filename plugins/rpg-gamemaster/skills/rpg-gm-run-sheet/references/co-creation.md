# Co-Creation (gm-run-sheet)

> Operational copy of the kit's shared DNA (master: `design-philosophy.md` at the kit root). This is the **front-end** of the skill — run it before assembling the run sheet. This skill is mostly mechanical assembly, so co-creation is light: confirm the inputs exist, confirm which session, then assemble.

> **About the examples here:** illustrative only — generic, or from public media (e.g. Critical Role's *Vox Machina*). **Never reuse them verbatim** — they are teaching aids; copying them would bleed unrelated flavor into the user's game.

---

## Part A — The co-creation loop

This skill needs **three things** before it can assemble a run sheet. They should already exist — the skill reads artifacts built by the other gamemaster skills, it does not create new fiction.

1. **The session-prep one-pager (or the latest `sessao` note)** — the session's shape: strong start, scenes, NPCs in play, revelations live this session.
2. **The clue map (`clue-map-*.md`)** — the revelation IDs (R1, R2…) and the placed clues, so the checklist can be keyed correctly.
3. **The active fronts and clocks** — the `frente`/`relogio` notes that supply the clocks dashboard and the stall backstops.

If any of these is missing, surface it before assembling. A run sheet built without the clue map cannot key its checklist; one built without active fronts has no clocks dashboard.

### Read the room first

- **User has just finished `rpg-session-prep`** → the one-pager is ready; ask which session slug to use as the filename, then assemble immediately.
- **User says "make me a run sheet for tonight"** → read the latest `sessao` note and the active `frente`/`relogio` state from the vault; confirm the clue map is present; assemble.
- **User provides the inputs directly** (pastes the one-pager or names the session) → confirm which clue-map revelations are live this session, then assemble.
- **"Just run with it"** → take what is in the vault, name your assumptions aloud ("I am using the latest sessao note and the two active fronts"), assemble the run sheet, and offer to tune.

State the **escape hatch early**: "Say 'just run with it' and I'll assemble the run sheet from what's in the vault right now."

### The loop — short for this skill

Because this is assembly rather than co-creation, the loop is tight:

1. **Confirm the inputs.** "I can see the session-prep one-pager, the clue map (R1–R5), and two active fronts. I will use those. Any corrections before I assemble?"
2. **Confirm the slug.** The output filename is `run-sheet-<slug>.md` — confirm the slug (usually the session name or date) if it is not obvious from the inputs.
3. **Assemble and offer to tune.** Produce the run sheet; invite the GM to mark any revelation as "not live this session" or swap an NPC.

**Give before you ask again.** This skill has almost nothing to elicit — do not interrogate. If you have what you need, assemble.

### The load-bearing decisions (the only things worth confirming)

1. **Which revelations are live this session** — pull from the session-prep one-pager; confirm if ambiguous.
2. **Which fronts/clocks to show** — default to all active fronts; ask only if there are more than four (to keep the run sheet to one page).

**Default everything else**: NPC quick-refs (derive from vault notes), stall backstops (derive from the next unfired portent on each clock), scene beat wording (compress from the one-pager).

### The test for "am I interrogating?"

If you have asked one question and not yet given anything back, you have already hit the limit for this skill. Assemble immediately after one round of confirmation. *Give before you ask again.*

## Part B — Read-aloud (inherited from the session plan)

The run sheet's read-aloud block is **inherited directly from the session-prep one-pager** — the strong start that `rpg-session-prep` already wrote. Do not write a new one. If the one-pager has no read-aloud block, derive a short sensory block from the "world's next move" beat in the active front.

Rules (from the kit's shared DNA, for reference when deriving):

- **~40–70 words.** Long enough to land the image; short enough to hold attention at the table.
- **Sensory and concrete.** What the players perceive right now — sight, sound, smell, the one wrong detail.
- **Never narrate the PCs.** Do not state what the characters feel, decide, or are about to do.
- **End open.** Trail off on something the players will immediately want to act on.
- **Mark it clearly:** a blockquote labelled **Read-aloud** (PT-BR: **Para ler à mesa**).
- **Match the user's language** in the entire output.

### Example (a strong start, ~50 words — generic)

*(Do not reuse verbatim. Adapt to the campaign's tone and the specific front that fired.)*

> **Read-aloud / Para ler à mesa:**
> *The warehouse district is quieter than it should be for midday. Crates stand unattended. The usual cluster of dock-hands at the corner post is gone. A single figure stands in the shadow of the loading bay — watching the street, not working it. One hand rests on something at their belt.*
